Entendiendo cómo funciona nuestro sistema de préstamos

La confianza se construye con claridad. Aquí explicamos detalladamente nuestros procesos, fórmulas y manejo de datos.

¿Por qué la Transparencia?

En nuestro compromiso por generar confianza y seguridad en nuestros usuarios, hemos desarrollado este espacio donde explicamos de manera clara y detallada:

  • Cómo funciona el sistema completo de préstamos
  • Las fórmulas matemáticas utilizadas en los cálculos
  • La relación entre los datos y cómo se interconectan
  • El flujo de información desde el registro hasta el pago final

Creemos que la transparencia es fundamental para construir relaciones duraderas basadas en la confianza mutua.

Arquitectura del Sistema

Componentes Principales:
  • Portal del Administrador: Registro y gestión de clientes, creación y administración de préstamos, registro manual de pagos, generación de reportes
  • Portal del Cliente: Visualización de préstamos activos, historial de pagos realizados, estado de cuenta en tiempo real, descarga de comprobantes
  • Base de Datos: Colecciones MongoDB para Clientes, Préstamos y Pagos

Flujo de Datos

Ciclo de Vida de un Préstamo:
  1. Registro del Cliente: Se recopilan datos personales
  2. Creación del Préstamo: Se define capital, plazo, tasa, etc.
  3. Registro de Pagos: El administrador registra pagos manualmente
  4. Cálculos Automáticos: El sistema calcula intereses, saldo, etc.
  5. Estado de Cuenta: El cliente visualiza en tiempo real

Fórmulas Matemáticas

Cálculos Principales:
  • Pago Esperado por Cuota: Pago Esperado = (Capital / Plazo) + Interés Programado
  • Interés Quincenal: Interés Quincenal = Capital × (Tasa de Interés / 100) × (1/24)
  • Saldo Actual: Saldo Actual = Capital Inicial - Σ(Abonos al capital)
  • Total Pagado: Total Pagado = Σ(Pagos)
  • Interés Acumulado: Interés Acumulado = Σ(Intereses Registrados)
  • Total a Pagar: Total a Pagar = Capital + Intereses + Atrasos
  • Diferencia: Diferencia = Total a Pagar - Total Pagado

Relaciones de Datos

Cliente → Préstamo: Un cliente puede tener múltiples préstamos

Préstamo → Pagos: Un préstamo puede tener múltiples pagos

Usuario → Acciones: Los administradores registran pagos y modifican datos

Seguridad y Privacidad

  • Autenticación JWT
  • Encriptación de contraseñas con bcrypt
  • Control de acceso por roles
  • Auditoría de acciones
  • Validación de datos

Compromiso con la Transparencia: Publicamos este documento para que nuestros usuarios comprendan exactamente cómo operamos. La confianza se basa en la claridad, no en la opacidad.
